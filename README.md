# Прогнозирование продаж (Ритмы продаж)

## 1. Датасет
Для обучения использован предоставленный файл `train.parquet`.
**Почему выбран этот датасет:** Данные содержат всю необходимую информацию для моделирования временных рядов на уровне товар-день: историю продаж (`qty`), ценовую политику (`price`, `is_promo`) и состояние запасов (`prev_leftovers`). Наличие анонимизированного идентификатора товара (`nm_id`) позволяет строить модели как на уровне пула товаров, так и с разделением по группам.

## 2. Список гипотез (до начала работы)

1.  **Лаговые признаки и скользящие окна:** Лаги продаж (1, 2, 3, 7, 14 дней) и скользящие средние (3-, 7-, 14-дневные) значительно улучшают прогноз, так как продажи имеют автокорреляцию.
2.  **Ценовые признаки и эластичность спроса:** Цена и её относительные изменения (изменение цены относительно среднего по товару) критически влияют на продажи — снижение цены повышает вероятность покупки.
3.  **Промо-акции и их отложенный эффект:** Признак `is_promo` и его лаги (была ли акция вчера, длина непрерывной цепочки промо) повышают продажи и улучшают прогноз.
4.  **Календарные и сезонные признаки:** Добавление календарных признаков (день недели, месяц, тригонометрическая сезонность sin/cos для дня года) улучшит метрику, т.к. спрос на многие товары цикличен.
5.  **Остатки товара (`prev_leftovers`):** Остатки на начало дня и их изменение помогают модели предсказывать отсутствие продаж (из-за отсутствия товара) и ограничивать максимально возможное количество в прогнозе.
6.  **Двухступенчатая модель (Zero-Inflated Sales):** Поскольку в данных много нулевых продаж, двухступенчатая модель — классификатор (продажа: да/нет) + регрессор (количество при продаже) — должна дать более низкую взвешенную MAE, чем единая регрессия.
7.  **Кластеризация товаров по паттернам продаж:** Разделение товаров на группы (сезонные, стабильные, умирающие) и добавление кластера как фичи позволит модели точнее улавливать общие паттерны внутри групп.

## 3. Краткие результаты

*   **Что подтвердилось:**
    *   **Лаговые признаки** оказались самыми важными. Модель без лагов продаж показывала значительно худший результат.
    *   **Ценовые факторы** (особенно отношение текущей цены к средней цене товара) вошли в топ-5 важности признаков в моделях градиентного бустинга.
    *   **Двухступенчатый подход (Zero-Inflated Model)** сработал эффективнее, чем сквозная регрессия, позволив точнее определять дни с нулевыми продажами и тем самым снизив итоговую wMAE.
    *   **Календарные признаки** (день недели) оказались значимы для товаров с выраженным выходным/будничным спросом.

*   **Что не подтвердилось / оказалось слабо полезным:**
    *   **Кластеризация товаров** не дала значительного прироста качества на валидации. Возможно, модель градиентного бустинга сама хорошо научилась разделять товары, либо количество кластеров было подобрано неоптимально.
    *   **Признак `prev_leftovers`** (остатки) сам по себе имел среднюю важность, но его взаимодействие с лагами продаж (например, были ли продажи вчера при нулевом остатке) оказалось полезнее, чем сырое значение.
    *   **Тригонометрическое кодирование дня года (sin/cos)** не улучшило метрику на коротком горизонте в 14 дней, так как сезонность внутри месяца оказалась важнее годовой.
